# 프로젝트 배경 및 설계 의도

> 작성일: 2026-07-10

## 해결하려던 문제

### 1. 실시간 데이터 동기화 지연
수백 명 규모의 대기 손님과 점포 스태프가 대기열 상태 변화(등록, 호출, 완료, 취소)를 지연 없이 동기화해야 하지만, 기존 폴링 방식은 서버 리소스를 과도하게 소비하고 실시간성이 떨어집니다.

### 2. 네트워크 불안정으로 인한 중복 요청
모바일 환경에서 연결이 일시적으로 끊기거나 지연될 때, 클라이언트의 재시도로 인해 동일한 대기 등록이 중복 생성되거나 순번 정합성이 깨질 수 있습니다.

### 3. 비정상 과다 요청 및 보안 위협
대기 등록 시 매크로 공격이나 비인가 API 호출로 서버 부하가 가중될 수 있으며, 비인가 사용자의 데이터베이스 직접 조작 위험이 존재합니다.

### 4. 비정형 대용량 데이터 관리
점포 상세 설정, 자주 변경되는 메뉴 구조, 일별 대기 데이터 통계화, 사업자등록증·메뉴 이미지 대량 업로드를 고속으로 처리하면서 데이터 정합성을 유지해야 합니다.

---

## 해결 방법

### 1. Goroutine 기반 SSE Broker 구축
무거운 WebSocket 대신, Go 채널과 Goroutine 기반의 경량 SSE 스트리밍 엔드포인트를 구현하여 대기열 상태 변경 감지 시 연결 중인 클라이언트에게 1초 이내로 단방향 Push 스트리밍합니다.

→ [SSE 구현 상세](./sse.ko.md)

### 2. 클라이언트 기반 UUID + 멱등성 검증 설계
클라이언트가 요청 시점에 `waiting_id`를 직접 생성하여 전송하고, 서버에서 DB와 대조하여 중복이면 기존 데이터를 그대로 반환함으로써 중복 등록을 근본적으로 방지합니다.

→ [멱등성 구현 상세](./idempotency.ko.md)

### 3. 미들웨어 기반 보안 레이어
`didip/tollbooth`로 IP당 초당 요청 수를 제한하고, Firebase Admin SDK로 모든 주요 백오피스 API의 JWT 토큰 유효성을 서버 측에서 검증합니다.

### 4. MongoDB Atlas + Cloudflare R2 조합
유연한 비정형 데이터 처리를 위해 MongoDB NoSQL을 채택하고, 이미지 등 무거운 파일 에셋 업로드는 S3 API 호환 스토리지인 Cloudflare R2에 위임하여 DB 서버 부하를 분산합니다.
