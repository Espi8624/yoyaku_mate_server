## Goroutineリークの防止

SSEストリーミング接続時に、クライアントの予期せぬ接続終了（`Context Done`）を即座に検知できない場合、Broker内部のGoroutineが破棄されずに累積していく問題が発生します。

`Context` と `Select` チャネルを活用したライフサイクル管理によってこれを防止します。

```go
for {
    select {
    case <-r.Context().Done():
        return  // クライアント接続終了を検知 → Goroutine終了
    case msg := <-clientChan:
        fmt.Fprintf(w, "data: %s\n\n", msg)
        w.(http.Flusher).Flush()
    }
}
```

## Rate Limiter しきい値の調整

モバイル/Webクライアントは、複数のアセットやAPIを同時に呼び出す特性があるため、単純に低いしきい値を設定すると、通常のユーザーもブロックされてしまいます。

- **現在の設定**: 5 req/s per IP, Burst 10
- Burst値によって一時的な並行リクエスト(画像ロードなど)は許容しつつ、継続的な過剰リクエストは遮断

## NoSQLの整合性と安全な例外処理

MongoDBドライバーのクエリ作成時に例外処理を漏れなく実装し、エラー変数を明確にキャッチして返却します。トランザクションのない単一ドキュメント書き込み操作の整合性は、バリデーション(Validation)で補完します。

## クライアント登録時刻の保護

サーバー到達時刻ではなく、 **クライアントが実際に「登録」ボタンを押した時刻** を `registration_time` として保存します。サーバー側でこれを上書きしないことで、ネットワーク遅延に関係なく、待機順序の時間的な整合性を保証します。
