# 기능 사양서: Response Time 대시보드 (Response Time Dashboard)

본 문서는 `yoyaku_mate_admin` 본사 관리자 웹에 구현된 API 레이턴시 실시간 모니터링 기능의 사양을 설명합니다.

> 작성일: 2026-07-20  
> 관련 문서: [구현 상세서: Response Time 대시보드](../implementation/response-time-dashboard.ko.md)

---

## 1. 개요 (Overview)

본사 관리자가 서버 API 응답 속도를 실시간으로 감시하고, 느린 엔드포인트를 즉각 식별할 수 있도록 하는 성능 모니터링 화면입니다.

단순 평균 레이턴시만 표시하는 것이 아니라, P95 / P99 퍼센타일 지표와 엔드포인트별 세분화 통계를 제공하여 **실제 운영 환경에서 병목 지점을 정확히 식별**할 수 있도록 설계되었습니다.

---

## 2. 시간 범위 선택 (Time Range Filter)

화면 상단의 토글 탭으로 집계 기준 시간을 전환합니다. 탭 변경 시 즉시 데이터가 갱신됩니다.

| 탭 | 집계 범위 | 용도 |
|---|---|---|
| **직근 5분** | 현재 시각 기준 최근 5분 | 실시간 장애 감지 |
| **직근 1시간** | 현재 시각 기준 최근 1시간 | 일반 운영 모니터링 (기본값) |
| **직근 24시간** | 현재 시각 기준 최근 24시간 | 일별 성능 추이 확인 |

---

## 3. 화면 구성 및 핵심 지표 (UI Metrics)

5초 주기 HTTP 폴링으로 자동 갱신됩니다.

### 3.1 상단 요약 카드 (Summary Cards)

선택된 시간 범위 내 전체 요청에 대한 집약 지표를 4개의 카드로 표시합니다.

| 카드 | 설명 | 정상 기준 |
|---|---|---|
| **AVG LATENCY** | 전체 평균 응답시간 (ms) | — |
| **P95 LATENCY** | 상위 5% 요청의 레이턴시 커트라인 | — |
| **P99 LATENCY** | 상위 1% 요청의 레이턴시 커트라인 | — |
| **ERROR RATE** | 4xx / 5xx 에러 요청 비율 (%) | 1% 미만 |

**색상 경보 기준:**
- 레이턴시 카드: `< 100ms` → 녹색 / `100~499ms` → 주황 / `≥ 500ms` → 빨강
- ERROR RATE 카드: `≤ 1%` → 녹색 / `1~5%` → 주황 / `> 5%` → 빨강

### 3.2 하단 엔드포인트별 레이턴시 테이블 (Endpoint Latency Table)

선택 시간 범위 내, 평균 응답시간 기준 내림차순 상위 10개 엔드포인트를 표시합니다.

| 컬럼 | 설명 |
|---|---|
| **METHOD** | HTTP 메서드 뱃지 (GET / POST / PATCH / DELETE) |
| **ENDPOINT** | API 경로 (monospace 폰트) |
| **AVG** | 평균 응답시간 (ms) |
| **P95** | 95th Percentile 레이턴시 |
| **P99** | 99th Percentile 레이턴시 |
| **REQUESTS** | 해당 기간 내 총 호출 횟수 |
| **ERROR %** | 해당 엔드포인트의 에러 요청 비율 |

각 레이턴시 값(AVG/P95/P99)은 임계치에 따라 색상이 동적으로 적용됩니다.

---

## 4. 모니터링 자체 요청 오염 방지

Admin 대시보드가 5초마다 폴링하는 `/api/admin/metrics/*` 경로는 `MetricsMiddleware`에서 `request_logs` 기록 대상에서 **명시적으로 제외**됩니다. 모니터링 요청이 모니터링 통계에 오염원으로 포함되는 것을 방지합니다.

---
