# 機能仕様書: システムメトリクスダッシュボード (System Metrics Dashboard)

本ドキュメントは、管理者用ダッシュボードの**システムメトリクス (System Metrics)** 機能に関する要件および仕様を定義します。

> 作成日: 2026-07-23  
> 対象: `yoyaku_mate_admin` (React UI), `yoyaku_mate_server` (Go Backend)

---

## 1. 概要 (Overview)
管理者がサーバー（fly.ioインスタンスまたはホスティング環境）のハードウェアリソース状態を直感的かつリアルタイムに把握し、障害の予防やサーバースケーリング（Scale-up/out）のタイミングを決定できるように支援するダッシュボード画面です。

## 2. 主な機能および要件 (Key Features & Requirements)

### 2.1 ハードウェアリソースのリアルタイム(Real-time)取得
- **取得項目**:
  1. **CPU Usage (CPU使用量)**: サーバー全体のCPU使用率 (%)
  2. **Memory Usage (メモリ使用量)**: サーバー全体の仮想メモリ使用率 (%)
  3. **Disk Space (ディスク使用量)**: ルート(`/`)パーティションのディスク使用率 (%)
- **データの性質**: 過去データの蓄積ではなく、API呼び出し時点の**現在のオペレーティングシステム(OS)**の状態値を返します。

### 2.2 ポーリング方式の5秒周期自動更新
- WebSocketやSSEの代わりに、実装がシンプルでサーバー負荷が少ない**5秒(5000ms)周期のポーリング(Polling)**方式を採用します。
- 管理者がダッシュボード画面に滞在している間のみ、バックグラウンドでデータを更新します。

### 2.3 直感的なUIの提供
- **Progress Bar**: 数値だけでなく、線形プログレスバー(Linear Progress)を通じて直感的に使用量を色別に区別して提供します。(例: CPU - Primary, Memory - Info, Disk - Warning色を使用)
- **エラーハンドリング**: 通信障害時に画面レイアウトが崩れず、自然にエラーメッセージを出力します。
