# 機能仕様書: リクエストダッシュボード (Request Dashboard)

本文書は、`yoyaku_mate_admin` 本社管理者ウェブに実装されたリアルタイムAPIトラフィックモニタリングシステム（リクエストカウンター）の機能仕様を説明します。

> 最終更新: 2026-07-14  
> 関連文書: [ADR-003: 自主メトリクス収集およびリクエストカウンターアーキテクチャ採用](../decisions/ADR-003-request-counter-architecture.md), [実装詳細書: リクエストカウンター](../implementation/request-counter.md)

---

## 1. 概要 (Overview)

プラットフォーム全体のAPIトラフィックフローと正常処理率を本社管理者ダッシュボードでリアルタイムに観測し、ボトルネックが発生しているAPIエンドポイントを識別してシステムの全般的な状態をひと目で把握するためのモニタリング画面です。

---

## 2. 画面構成要素

画面の上部には主要なメトリクスカード3つが横並びで表示され、下部には個別のリクエストローデータ一覧を提供します。5秒周期で自動更新（HTTP Polling）されます。

### 2.1 上部サマリーカード (Metrics Cards)
* **TOTAL REQUESTS (24H)**: 直近24時間以内にサーバーへ流入した全HTTP APIリクエスト数です。
* **SUCCESS RATE**: 24時間の全リクエストのうち、正常応答（HTTPステータスコードが2xxおよび3xx系）を返した成功率（%）指標です。エラー応答（4xx/5xx）が増加するにつれてリアルタイムに成功率が減少します。
* **PEAK TPS (1H)**: 直近1時間で最もトラフィックが集中した秒間最大リクエスト処理量（Transactions Per Second）です。

### 2.2 リアルタイムAPIリクエストログテーブル (Latest Logs Table)
直近で発生した50件のAPIリクエスト履歴を最新順でソートしてグリッド形式で表示します。
* **表示列**: 日時（Timestamp）、HTTPメソッド、APIパス、ステータスコード（Status Code）、応答速度（Latency）、クライアントIP。
* **視覚的強調機能**:
  * **エラー行のハイライト**: ステータスコードが400系（Bad Requestなど）の行はオレンジ色のティント、500系（Internal Error）の行は赤色のティントで背景をハイライトし、障害状況を即座に検知できるよう支援します。
  * **応答遅延時間の警告**: APIの応答速度が500ms以上〜1000ms未満の場合はオレンジ色の太字、1000ms以上の場合は赤色の太字でテキストを強調し、応答遅延区間を追跡します。
