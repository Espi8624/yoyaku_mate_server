# 기능 사양서: 에러 대시보드 (Error Dashboard)

본 문서는 `yoyaku_mate_admin` 본사 관리자 웹에 구현된 실시간 에러 감시 및 시스템 상태 모니터링 기능의 사양을 설명합니다.

> 최종 수정: 2026-07-14  
> 관련 문서: [ADR-002: 에러 대시보드 내 HTTP 폴링 방식 채택](../decisions/ADR-002-use-polling-for-error-dashboard.ko.md), [구현 상세서: 에러 대시보드](../implementation/error-dashboard.ko.md)

---

## 1. 개요 (Overview)

본사 관리자가 플랫폼 전체의 시스템 안정성 및 장애 상태를 즉각적으로 인지할 수 있도록, 서버 내부 오류 통계와 에러 발생 이력을 실시간으로 감시하고 트래킹하는 화면입니다.

---

## 2. 화면 구성 및 핵심 지표 (UI Metrics)

관리자 백오피스의 전용 대시보드로 노출되며, 5초 주기로 자동 갱신(HTTP Polling)됩니다. 반응형 웹 디자인을 지원합니다.

### 2.1 상단 에러 메트릭 요약 카드 (Metrics Cards)
총 4가지 에러 카테고리에 대한 누적 집계값을 가로 그리드로 보여줍니다.
* **500 ERRORS**: 고 내부 패닉, 컴파일 런타임 등 500 계열 서버 내부 실패 횟수
* **400 ERRORS**: 잘못된 요청 인수 전송, 존재하지 않는 404 경로 호출 등 400 계열 실패 횟수
* **DB ERRORS**: MongoDB 접속 유실, 쿼리 구문 예외 등 데이터베이스 영역 에러 발생 횟수
* **SSE DISCONNECTS**: 대기열 실시간 중계 중 브라우저 탭 닫기 등으로 강제 유실된 SSE 연결 횟수

### 2.2 하단 최근 에러 로그 테이블 (Latest Logs Table)
가장 최근에 백엔드 DB에 적재된 에러 로그 데이터 최대 50건을 최신 정렬로 노출합니다.
* **그리드 컬럼**:
  * **일시**: 에러 발생 시각 (사용자의 로컬 디바이스 타임에 맞춰 자동 변환)
  * **타입**: 에러의 대분류 식별 코드 (예: `400_BAD_REQUEST`, `500_INTERNAL_ERROR`, `DATABASE_ERROR`, `SSE_DISCONNECT`)
  * **API경로**: 호출을 유발한 API 메소드 및 경로 (예: `GET /api/admin/metrics`)
  * **메시지개요**: 발생 에러의 요약 메시지 (지정 길이를 초과할 경우 생략 부호 처리)
* **상세 팝업 모달**:
  * 테이블 행을 마우스로 클릭하면, 문제 해결을 돕는 에러 상세 스택 트레이스 등을 별도의 레이어 모달창으로 팝업 제공합니다.
